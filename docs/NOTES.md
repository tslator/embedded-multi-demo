# Pi Pico
## Pico Getting Started
I followed the "Getting Started" guide for the pico.  Mostly went as expected.
- After installing the picotool for windows using ZaDig installer, it is necessary to restart vscode
- Vscode does not handle multiple projects within a workspace.  Consequently, to get the pico extension to recognize a pico project it is necessary to open the pico project folder in a separate instance of vs code.  This is a known issue with vscode and I have experienced it with STM32 as well.
